Fix the root metadata title and verify current mock data against locked data model shapes —

## Files to Modify

### 1. `src/app/layout.js`

**Current problem:**
Line 15 reads `title: "Create Next App"` — the default Next.js scaffold title.
Every page in DineFlow is currently serving this title to browsers and search engines.

**Required changes:**
- Change `title` from `"Create Next App"` to `"DineFlow"`
- Change `description` from `"Generated by create next app"` to `"Michelin-service restaurant CRM and operations suite"`
- Do not touch anything else in this file — fonts, body className, and html lang attribute must remain unchanged

**Constraint:** This file is a Server Component. Do NOT add `"use client"` to it.

---

### 2. `src/app/(crm)/orders/page.jsx` — Data Shape Fix Only

**Current problem:**
The mock data array `initialActiveOrders` uses `timer: "18m ago"` and `timer: "12m elapsed"`
as static strings. Per the data model in `AGENTS.md`, the `Order` shape requires a real
`createdAt: Date` timestamp — not a string label.

**Required changes:**
- Replace every `timer` field in `initialActiveOrders` with a `createdAt` field
- `createdAt` must be a real JavaScript `Date` object computed relative to `new Date()`
  - Example: an order that arrived 18 minutes ago → `new Date(Date.now() - 18 * 60 * 1000)`
- Do NOT yet remove this mock data from the page file — that is Phase 3's job
- Do NOT yet add any timer display logic — that is Phase 4's job
- The only goal: the shape of each mock object must match the locked data model exactly

**Constraint:** The `timer` key must be completely removed and must not exist alongside `createdAt`.

---

### 3. `src/app/(crm)/service/page.jsx` — Comment Only, No Logic Changes

**Current problem:**
The mock data array `initialOrders` uses `time: "2m ago"` and `time: "5m ago"` as display
strings. This page's order shape is a simplified view-model, not the full canonical Order —
it uses `title` and `subtitle` instead of an `items` array. This is acceptable for now since
Phase 3 will replace this data entirely with the shared store.

**Required change:**
- Add a comment above each `time` field: `// TODO Phase 3: derive from createdAt in ordersStore`
- Do not restructure this page's data — Phase 3 will replace it entirely

## Architecture Rules That Apply to This Phase

From `AGENTS.md`:

- **Rule 3 (No Duplicate Font or Style Imports):** Fonts live only in `globals.css`. If any
  font imports are found in page files during the verification step, remove them.
- **Rule 3 (No Meta in JSX):** If any `<meta>`, `<title>`, or `<link>` tags remain in page
  JSX, remove them. Page metadata uses `export const metadata = {}` only.
- **Data Model Rule:** `Order.timer` does not exist in the locked model. `Order.createdAt`
  is a `Date`. Enforce this in the orders page mock data.
